Emplacement et objectifs

Le Lore de l'empire Nexuria est hébergé sur le repo "dytopia-wiki", du projet "Monde Dytopia", toute la structure des pages sont raccordés dans le dossier "nexuria-internet".
Toutes les informations du Lore sont placer sous un autre repo accesible et contribuable, appelé "explorateur-nexuria".

Toutes la structure des informations présentes dans les repos cités dans "Emplacement et objectifs" sont sous la propriétée de KishiNight Production et sous license CC BY-NC-ND 4.0. (Voir "Mentions légales".)

Emplacement de contribution

L'emplacement pour contribuer au Lore de Nexuria n'est pas dans le repo nommé "dytopia-wiki", mais dans celui nommé "explorateur-nexuria".
Chaque contenu doit être rédigée en markdown et les règles de rédaction seront précisées dans "Règles de rédaction".
Enfin, pour sa contribution vous devrai utiliser Github (Voir la suite).


Création de votre contribution

Pour pouvoir contribuer correctement il n'as qu'à ce jour une seule solution. S'inscrire sur Github et créer un fork depuis le repo "explorateur-nexuria".
Seulement les ajouts de vos documents ou les modifications de seulement vos documents sont autorisées. La modification de d'autres documents ne vous appartenant pas, devra être d'une justification neutre et justifiable.

Pour les nouveaux, nous avons créer un guide de la création d'un fork à la création d'une PR (pull request), afin de mettre à jour votre contribution sur le repo principal.

Créer un fork

Pour accéder au repo "explorateur-nexuria", vous pouvez cliquer ici.
Par la suite, vous pouvez suivre les étapes via les images ci dessous ou chercher par vous même:

  1. Fork
  2. Create a new fork
  3. Remplir la page de création de fork
    • Repository name (pas de noms abusif ou troll).
    • Description (recommender: garder l'ancien).
    • Copy the "main" branch only (laissez cocher).
  4. Create fork

Ajouter vos documents

Après la création de votre fork il est essentiel d'ajouter vos documents. C'est pour ceci qui faudra rédiger les fichiers en markdown.
Pour l'ajout à votre fork, utiliser VS code ou bloc notes pour créer votre premier fichier. Par la suite vous pourrez le télécharger depuis votre fork. Et n'oubliez pas de commit vos téléchargements en précisant vos modifications ou ajouts.

Il existe d'autres moyens, celle ci est recommender pour ceux commençant Github.

Faire un pull request

Sur Github, vous avez la possibilité de faire joindre vos modifications et ajouts sur le repo principal par "pull request", pour ce faire, il est essentiel d'être sur votre fork pour créer votre pull request.
Pour accéder à l'emplacement des pull request, sélectionner "pull request" (voir images), et créer en un nouveau qui doit obligatoirement être relier à la branche principale, sois la branche "main".

Il est important que votre description du pull request doit contenir les justifications de modification d'un ou des documents ne vous appartenant pas, ou l'ajout de nouveaux documents précisés dans les règles de rédaction.

Règles de rédaction

Avant toutes choses, chaques documents intégrés au Lore doit avoir en tête ceci: Un document de l'empire Nexuria, représente une partie même de l'histoire par un auteur à l'intérieur de l'empire.

Pour ceux n'ayant pas compris cette phrase, lorsque vous écrivez un document vous devez être un auteur interne, et non quelqu'un d'extérieur au monde et qui la regarde.
Ce n'est pas une simple description, mais c'est des actions, une histoire, des intégrations, qui peut mener à une description selon le type d'un document.

Avant de rédiger

Avant de rédiger, il faut comprendre ce qu'on va écrire. Parce que écrire un journal informatif ou l'enrengistrement d'une personne écrit par texte, son ton, ces contextes et encore, ne seront pas les mêmes.
C'est pour ceci qu'on vous demande de réfléchir à comment vous allez composer vos documents avec son ou ces contextes, son rapport, ou le ton d'écriture qui doivent être logique et cohérente.

Lorsque vous aurez votre idée, votre document doit répondre à ces critères (préciser dans la description du pull request pour chaques documents):

  • Qu'est qui mène à ce document ? (Contexte, Nécessité pour le document)
    • Exemple: La création du haut conseil.
    • Exemple: L'invention d'une formule mathématique.
    • Information: Le ou les contextes doivent être obligatoirement écrite dans le Lore en respectant ces mêmes règles (un lien de redirection vers le ou les documents).
  • Qui écrit ce document ? (Auteur, Hiérarchie)
    • Exemple: Dr. Jaipasdenom, chercheur en chef de l'institut de recherche impériale.
    • Exemple: M. Quelquun, Xème président de l'empire.
    • Information: Les informations sur le personnage doit être écrit dans d'autres documents (un lien de redirection vers le ou les documents).
  • Pourquoi doit t'on écrire ce document ? (Raison, utilitée)
    • Exemple: Rapport sur l'activité d'une explosion grave.
    • Exemple: Contextualisé de l'histoire.
Vous pouvez toujours voir les anciens pull request du repo pour vous aider à comprendre et avoir des exemples.

Lors de la rédaction

Le format de votre document est libre, sauf dans les cas où un format est déjà utilisé, comme la description d'un personnage dans les dossiers des citoyens et encore.
Sinon ce changement de format devra être décrit dans un autre document.

Par rapport au code mardown: ce code à pour particularité d'être très simple à comprendre. Mais avant de continuer la rédaction, au début de chaque document, il est obligatoirement de mettre "une description" de votre fichier.
Les possibilités à placer sont:

  • nom du fichier
  • date (celui du Lore, modification ou création. Il est important que la logique du temps soit respecté, sauf si il est justifié correctement.), date-fichier (la création de votre fichier avec la vrai date IRL)
  • date-fichier (la création de votre fichier avec la vrai date IRL)
  • tags (accès au fichier voir la liste)

Nomanclature des noms de fichiers

N'oubliez pas que chacun des documents doit avoir cette nomenclature comme nom de fichier hors celui intégré au code: Document-AAA-BBB-CCC-DDD-EEE-ETC.md, où A est le sujet principal, B le sujet secondaire, C le sous sujet secondaire, D la numérotation du sujet, E la sous numérotation du sujet et encore.

Exemple: Document-NEX-TECH-BREV-01.md
-> NEX: Pour Nexuria
-> TECH: Pour Technologique
-> BREV: Pour un brevet
-> 01: Pour le numéro du brevet

Exemple de code:

--- #Commencement de la description
title: nom-du-fichier
date: date-lore
date-fichier: date-IRL
tags: tags-accès #Peut être vide
--- #Fin de la description

Code markdown

Si vous avez besoin d'utiliser un code markdown en particulier, merci de nous le faire savoir. Nous utilisons une liste restreinte du code markdown pour éviter les mauvaises surprises.

Avec les "#"

# Contenu: Titre de niveau 1

Exemple:

Contenu

## Contenu: Titre de niveau 2

Exemple:

Contenu

### Contenu: Titre de niveau 3

Exemple:

Contenu

Avec les "*"

*Contenu*: Italique

Exemple:

Contenu

**Contenu**: Gras

Exemple:

Contenu

***Contenu***: Gras + Italique

Exemple:

Contenu

Avec les "`"

`Contenu`: Code

Exemple:

Contenu

while a > 0:
    return True

: Bloc de code

    Nos et vos droits

    Il est important de savoir qu'elle sont vos droits par rapport à vos documents ou contenus que vous intégrer au Lore.

    Nos droits

    D'abord, nous mettrons tout en oeuvre pour que nos mentions légales soient respectées. C'est à dire que, d'autres contenus ayant une license, qui est intégrés au Lore, nous avons la possibilité de la supprimer.
    Cela concerne seulement du contenu utiliser dans le même contexte et sujet, comme un nom, une invention, et bien d'autres. Mais si la license nous le permet, nous pourrons garder le contenu.

    Ensuite, comme vous pouvez le savoir, nous possédons les droits sur le contenu intégré dans ce projet, c'est à dire que nous pouvons supprimer, proposer, modifier, du contenu seulement si nous le jugons nécessaire dans le cadre d'une incohérence, d'un contenu illégal, ou des problèmes de droits d'auteur.
    Lorsqu'un contenu est supprimer ou modifier, notre équipe de rédaction s'assureront que la raison est accesible par l'auteur du contenu.

    Vos droits

    Lorsque vous publiez votre contenu sur notre Lore, ce contenu vous appartient, vous avez le droit de demander la suppresion, ou de le supprimer par vous mêmes. Cette limite est atteinte si le contenu doit rester pour la cohérence du Lore.
    Vous pouvez tout de même retirer vos droits sur le contenu.

    Par les règles de rédaction, vous pouvez modifier d'autres contenus, pour des raisons de syntaxe, de cohérences, ou de manque de détails, cela ne changera en aucun cas les droits sur le contenu lié, il reviendra toujours à son créateur.
    Pour certains contenus, nous pouvons demander que le créateur original soit d'accord pour une modification ou suppresion de contenu. Ce droit est réservé si cela consiste à une grosse modification qui change le sens du contenu original.
